# Diagramatix — Quick Reminder Sheet 3

## Flowchart Summary

Flowcharts describe a procedure as a sequence of steps and decisions using the monochrome ISO 5807 symbol set. Flow runs top-to-bottom (or left-to-right) along arrows; a decision splits it and connectors rejoin it. Below is each symbol on the Diagramatix flowchart palette and when to reach for it.

**Terminator** — A stadium (rounded-end) shape marking the **Start** or **End** of the procedure. Place one at every entry and exit point.

**Process** — A plain rectangle for a task, action, operation, or calculation. The default building block — one box per step.

**Decision** — A diamond posing a yes/no or multi-way question; each answer leaves on a labelled branch. Use it wherever the path depends on a condition.

**Input / Output** — A parallelogram for data entering or leaving the process. Use it for reading inputs and producing outputs.

**Document** — A rectangle with a wavy bottom edge for a single document produced or consumed. **Multiple Documents** — a stacked variant for a document pack or several documents.

**Predefined Process** — A rectangle with double side-bars naming a process defined elsewhere (a sub-routine). Use it to call out reusable, separately-documented procedures.

**Preparation** — A hexagon for setup, initialisation, or configuring a value before the main steps run.

**Manual Input** — A shape with a sloped top for data a person keys in. **Manual Operation** — an inverted trapezoid for work a person performs without automation.

**Display** — A curved "screen" shape for information shown to a user. Use it when output is presented on-screen rather than stored.

**Delay / Wait** — A D-shape for a deliberate pause or waiting period in the flow.

**Database** — A cylinder for structured electronic data held in a database or persistent store.

**On-page Connector** — A small circle that continues the flow elsewhere **on the same page**, avoiding crossing lines. **Off-page Connector** — a pentagon that continues the flow **on another page**.

**Merge** — A downward triangle that combines several incoming flows into one. **Parallel (fork/join bar)** — a bar where concurrent branches split apart or re-synchronise (flip it for a vertical layout).

**Comment** — A rounded rectangle holding a note, attached to an element by a dotted association line for explanation.

**Swimlane** — A vertical column that groups steps by role or department; drop another to add a column to the right.

---
*Flowcharts in Diagramatix are strictly monochrome by design — structure and labels carry the meaning, not colour.*
